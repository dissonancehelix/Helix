# system/

Machinery lives here.

- `engine/` is the trust kernel: schemas, checks, contracts, validators, and enforcement code.
- `tools/` is runnable machinery: pipelines, bridges, diagnostics, importers, exporters, and local utilities.
- `vendor/` is cloned or mirrored third-party code and SDK material.

`system/` enables Helix, but it does not own the profile. Canon remains `DISSONANCE.md` plus `core/map/`.
