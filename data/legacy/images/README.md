# Image Evidence

Status: local raw evidence preserved under `data/raw/images/`.

This area is for loose image intake and compact visual-evidence notes. It is not a first-class domain. Images become useful to Helix when they clarify a reusable pattern, strengthen an example, or provide a fixture for a future application.

Raw images are intentionally not tracked in Git by default. Keep source files local under `data/raw/images/`; track only summaries, manifests, pattern links, or deliberately curated fixtures.

## Re-analysis

The recurring signal is not a single demographic type. It is channel convergence with low internal friction: face, posture, styling, self-presentation, softness/sharpness balance, and context all need to feel mutually authored.

Observed pattern handles:
- `attraction_channel_convergence`: attraction strengthens when several channels agree without feeling staged into a single flat type.
- `signal_ownership`: the image works better when the person appears to own the signal rather than being overwhelmed by it.
- `private_edge_requirement`: a slight interior/private edge makes presentation feel more live and less billboard-like.
- `soft_base_sharp_event`: soft visual base with sharper events in eyes, hair, silhouette, jewelry, makeup, pose, or clothing.
- `bounded_abundance`: variation is welcome when the presentation remains coherent.

Best compressions:

> Attraction is channel convergence with low internal friction, not demographic type-detection.

> The signal needs an owner.

> Beauty here often has a private edge, not just public polish.

## Preservation Rule

Do not commit broad raw image dumps unless a future visual-corpus application needs reproducible fixtures. Promote only selected, rights-safe, deliberately named examples when they become necessary for a tool, report, or model check.
