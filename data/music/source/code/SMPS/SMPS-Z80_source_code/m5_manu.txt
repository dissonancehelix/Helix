	　以下に各データ形式の書式を簡略に記します。詳細は２章を参照してください。

　１−Ａ：音長データ
	　直値、またはアサインを用います。例えば
            ４分音符		Ｌ４
            符点４分音符	ＬＦ４
            ３連符		Ｌ４＿３

　１−Ｂ：音程データ
	①音符ＮＯＲＭＡＬモード
	　書式： 音符データ，音長データ，音符データ，音長データ

	　音符データは、アサインで、Ｃ０からＢＳ７まで８オクターブあります。
	　  例：　DB   	C0,L4,E0,LF8
	  前のデータと音程または音長が同じ時、省略することができます。
	    例：　DB	C0,L8,D0,L8,E0,L4 → DB   C0,L8,D0,E0,L4
	    例：　DB	C0,L8,C0,L4,E0,L4 → DB   C0,L8,L4,E0
	　ただし、休符のときは音長のみ省略できます。
	    例：　DB	NL,L1,NL,L1,NL,L8 → DB   NL,L1,NL,NL,L8

	②音符ポルタメントモード
	  書式： 音符データ，符号付変化量，音長データ

	　音程、音長共に、省略できません。
	    例：　DB	C0,0F0H,L8

	③ダイレクトＦＲＥＱ　ＮＯＲＭＡＬモード
	　書式： ＦＲＥＱデータ　ＨＩＧＨ，ＬＯＷ，音長

	　音程、音長共に、省略できません。
	　　例：　DB	0,0F0H,L8

	④ダイレクトＦＲＥＱ　ポルタメントモード
	　書式： ＦＲＥＱデータ　ＨＩＧＨ，ＬＯＷ，変化量，音長データ

	　音程、音長共に、省略できません。
	　　例：　DB	0,0F0H,020H,L8

　１−Ｆ：音色データ
	　２５バイトで１音色を構成します。つぎの並びになっています。
	
	
	０ｂｙｔｅ　　	：	 bit 0-2 = algorithm   / 3-7 = feedback  

	１		：slot 1 bit 0-3 = multiple    / 4-7 = detune
	２		：slot 2 bit 0-3 = multiple    / 4-7 = detune
	３		：slot 3 bit 0-3 = multiple    / 4-7 = detune
	４		：slot 4 bit 0-3 = multiple    / 4-7 = detune

	５		：slot 1 bit 0-4 = attack rate / 6-7 = key scale
	６		：slot 2 bit 0-4 = attack rate / 6-7 = key scale
	７		：slot 3 bit 0-4 = attack rate / 6-7 = key scale
	８		：slot 4 bit 0-4 = attack rate / 6-7 = key scale

	９		：slot 1 bit 0-4 = decay rate  / 7 = am on
　　　１０		：slot 2 bit 0-4 = decay rate  / 7 = am on
　　　１１		：slot 3 bit 0-4 = decay rate  / 7 = am on
　　　１２		：slot 4 bit 0-4 = decay rate  / 7 = am on
		
　　　１３		：slot 1 bit 0-4 = sustain rate
　　　１４		：slot 2 bit 0-4 = sustain rate
　　　１５		：slot 3 bit 0-4 = sustain rate
　　　１６		：slot 4 bit 0-4 = sustain rate

　　　１７		：slot 1 bit 0-3 = sustain level / 4-7 rerease rate
　　　１８		：slot 2 bit 0-3 = sustain level / 4-7 rerease rate
　　　１９　		：slot 3 bit 0-3 = sustain level / 4-7 rerease rate
　　　２０		：slot 4 bit 0-3 = sustain level / 4-7 rerease rate

　　　２１		：slot 1 bit 0-6 = total level / 7 carier flag
　　　２２		：slot 1 bit 0-6 = total level / 7 carier flag
　　　２３		：slot 1 bit 0-6 = total level / 7 carier flag
　　　２４		：slot 1 bit 0-6 = total level / 7 carier flag




　１−Ｇ：ビブラート、エンベロープデータ

	　Ｍ５ＴＢ１３．ＳＲＣにエンベロープテーブル、ビブラートテーブルが
	あります。各テーブルとも符号付きデータとなっています。
	　ビブラートは、サウンドルーチンが呼び出される毎に、曲またはＳ．Ｅ
	のテーブルデータによって割り出された分周データに対して、テーブルデ
	ータがたしあわされます。ＦＭ／ＰＳＧともに有効です。
	　エンベロープは、サウンドルーチンが呼び出される毎に、ＶＯＬＭデー
	タに対して、テーブルデータがたしあわされます。ＰＳＧのみ有効です。
	
		書式：

			LAVEL:
				DB	<data>,<data>, .... ,<table command>
		
	　なお、テーブルコマンドには、つぎのものがあります。


	***** ビブラート，エンベロープテーブル　コマンド *****


	ＴＢＲＥＰＴ		テーブルの先頭に戻る。

　２−Ｂ：音程データ
	　音符モードにおいて、音長、音符、コマンドはつぎのように割り振っています。
		音長		１−７ＦＨ
		音符		８０Ｈ−ＤＦＨ
		コマンド	Ｅ０Ｈ−ＦＦＨ


	①音符ノーマルモード
	　書式： 音符データ，音長データ，音符データ，音長データ

	　音符データは、Ｃ０からＢＳ７まで８オクターブあります。

	　Ｃ０はＰＳＧ周波数データで８５４Ｈ、ＦＭで２８４Ｈとしています。

	　移調はＢＩＡＳデータにより行うことができます。ＢＩＡＳデータは１が半音にあたります。符号付きデータなので、０ＦＦＨの場合半音低い音がでます。


	②音符ポルタメントモード
	  書式： 音符データ，符号付変化量，音長データ
	    例：  DB	C3,-1,6

	　このモードでは初期音程から始まり、サウンドルーチンがＣＡＬＬされる毎に変化量を前の音程に足し合わせて出力する操作を音長分繰り返します。

	　テーブルデータが
		　DB    C0,010H,4
	のとき、実際のデータは
		PSG FREQ    　FM FREQ
          1:      854H          284H
	  2:      864H	        294H
	  3:      874H          2A4H
	  4:      884H          2B4H
        となります。

	　ＰＳＧはデータが大きいほど低い音になるので、この場合音程は下がります。ＦＭの場合その逆で上がります。

	③ダイレクトフリケンシーノーマルモード
	　書式： ＦＲＥＱデータ　ＨＩＧＨ，ＬＯＷ，音長

	　テーブルのＦＲＥＱデータとＢＩＡＳ値を足して音程としてチップにセットします。

	　ＢＩＡＳが０、テーブルデータが
		　DB    03H,050H,4
	　のとき、実際の出力データは
	          1:     350H
		  2:     350H
		  3:     350H
		  4:     350H
	　となります。


	④ダイレクトフリケンシーポルタメントモード
	　書式： ＦＲＥＱデータ　ＨＩＧＨ，ＬＯＷ，変化量，音長データ

	　このモードでは、初期音程（テーブルデータ＋ＢＩＡＳ値）から始まり、サウンドルーチンがＣＡＬＬされる毎に変化量を前の音程に足し合わせて出力する操作を音長分繰り返します。

	　テーブルデータが
		　DB    2,0F0H,010H,4
	のとき、実際のデータは
			 FREQ
        	  1:     2F0H
		  2:     300H
		  3:     310H
		  4:     320H
       となります。


　２−Ｃ：リズムデータ

	　ＦＭ及びＰＳＧのデータは　Ｍ５ＦＭＤＲ１３．ＳＲＣ内にあるテーブル
	（ＦＭは ＤＲＴＢ,ＰＳＧはＰＳＧ＿ＤＲＴＢ）から曲データにしたがって逐次呼び出されます。

	　ＤＲＵＭ ビットアサインは次のようになっています。

			7 6 5 4 3 2 1 0
			| |     |--------- FM RYTHM NO.
			| |
			| |----------------PSG RYTHM NO.
			|------------------RYTHM FLAG

	　したがって、ＦＭのリズムは１５種（１−１５）、ＰＳＧは７種（１−７）もつことができます。

		　ＡＵＴＯＰＡＮには３つのモードがあります。ＰＡＮ　ＮＯ．で設定します。

	　　モード１：（ＰＡＮ　ＮＯ．＝１）
	
		　まず、＜ＰＡＮ　ＴＡＢＬＥ　ＮＯ．＞に従って＜ＰＲＪ＞ＣＮＴ．ＳＲＣのｐａｎ＿ａｄｄｒ＿ｔｂをスキャンします。ＮＯ．が０の場合ｐａｎ＿０＿ｄａｔａのテーブルにしたがってＬ／Ｒの設定が行われます。＜ＰＯＩＮＴ　ＳＴＡＲＴ　ＮＯ．＞によって、テーブルの読みだし開始位置を決定します。０の場合、トップから、１の場合１バイト後からになります。そして、＜ＬＥＮＧＴＨ　ＤＡＴＡ＞分カウントダウンした後、つぎのテーブルデータを読みだしセットします。このモードのカウント単位は発音数です。したがって＜ＬＥＮＧＴＨ　ＤＡＴＡ＞音出力した後、つぎのＬ／Ｒがセットされます。なお、ＴＩＥコマンドでつなげられているデータは、１音とカウントします。テーブルデータをテーブルトップから＜ＰＯＩＮＴ　ＬＩＭＩＴ＞バイト分読み出したところで、テーブルのＴＯＰから再度データを読みだします。このとき、＜ＰＯＩＮＴ　ＳＴＡＲＴ　ＮＯ．＞は無視されます。

		　次に例を示します。
		　テーブルデータが
			DB	AUTOPAN,1,2,1,3,2
			DB	C3,L8,D3,E3,F3,G3,A3,B3
		のとき、＜ＰＡＮ　ＴＡＢＬＥ　ＮＯ．＞は２なので、ＰＡＮ＿２＿ＤＡＴＡのテーブルよりデータを読みだします。
			pan_2_data::
				db	0c0h
			pan_3_data::
				db	040h
				db	080h
				db	0c0h
		　また、＜ＰＯＩＮＴ　ＳＴＡＲＴ　ＮＯ．＞が１なので、テーブルＴＯＰから１バイト後ろの４０Ｈが設定され、Ｃ３の音がＲから出ます。＜ＬＥＮＧＴＨ　ＤＡＴＡ＞が２なので、Ｄ３の音までの２音がＲから出ます。Ｅ３、Ｆ３の音は、テーブルＴＯＰから２バイトめの８０Ｈが設定され、Ｌから出ます。Ｇ３、Ａ３の音は、テーブルＴＯＰから３バイトめのＣ０Ｈが設定され、Ｌ／Ｒから出ます。ここで、テーブルＴＯＰ＋＜ＰＯＩＮＴ　ＬＩＭＩＴ＞までのデータを設定したので、次のデータの読みだしは、テーブルのＴＯＰに戻り、Ｃ０Ｈが設定されます。

		Ｂ）　＜ＰＯＩＮＴ　ＬＩＭＩＴ＞後は、ループせずそのままの出力を保つ
		Ｃ）　つぎの音を発音するときにイニシャライズされることです。
