1: Data format
	The data is segregated as follows

	A: Sound length data
		Normal
		Note data
		Portamento

	B: Note data
		Direct
		Normal
		Frequency
		Data
		Portamento

	C: Rhythm data

	D: Command data

	E: Header data

	F: Tone data

	G: Vibrato and envelope data

	 The format of each data format is briefly described below. For more information, please refer to chapter 2.



　1-A: Note length data
	　Use direct values or assignments. For example
            quarter note         L4
            Dotted quarter note  LF4
            Triad                L4_3



　1-B: Pitch data
	1) Note NORMAL mode
	　Format: note data, note length data, note data, note length data

	　The note data has 8 octaves from C0 to BS7 by assignment.
	　  Example: DB C0,L4,E0,LF8
	  This data can be omitted if it has the same pitch or duration as the previous data.
	    Example: DB C0,L8,D0,L8,E0,L4 → DB C0,L8,D0,E0,L4
	    Example: DB C0,L8,C0,L4,E0,L4 → DB C0,L8,L4,E0
	　However, only the note length can be omitted for rests.
	    Example: DB NL,L1,NL,L1,NL,L8 → DB NL,L1,NL,NL,L8

	2) Note portamento mode
	  Format: note data, signed variation, note length data

	　Both pitch and duration cannot be omitted.
	    Example: DB C0,0F0H,L8

	3) Direct FREQ NORMAL mode
	　Format: FREQ data HIGH, LOW, note length

	　Both pitch and note length cannot be omitted.
	　　Example: DB 0,0F0H,L8

	4) Direct FREQ Portamento mode
	　Format: FREQ data HIGH, LOW, amount of change, note length data

	　Both pitch and note length cannot be omitted.
	　　Example: DB 0,0F0H,020H,L8



　1-C: Rhythm data
        　With the following assignments, it is possible to play FM and PSG simultaneously.

	*FM
		S  Snare
		HT Hi Tom
		MT Mid Tom
		LT Low Tom
		B  Bass Drum
		H  HiHat

	*PSG
		PH  PSG HiHat
		PO  PSG Open HiHat

	*FM + PSG
		OS  PSG Open  + FM Snare
		HS  PSG HiHat + FM Snare
		HB  PSG HiHat + FM Bass Dr.
		OB  PSG Open  + FM BassDr
		POH PSG Open  + FM HiHat


	　These combinations represent the rhythm data.

	　Format: Rhythm data, note length data
		(Like note data, it can be abbreviated.)
	　　Example: DB BH,L8,H,SBH,H

　1-F: Voice data
	　25 bytes make up one tone. The order is as follows
	
	
	０ｂｙｔｅ　　	：	 bit 0-2 = algorithm   / 3-7 = feedback  

	１		：slot 1 bit 0-3 = multiple    / 4-7 = detune
	２		：slot 2 bit 0-3 = multiple    / 4-7 = detune
	３		：slot 3 bit 0-3 = multiple    / 4-7 = detune
	４		：slot 4 bit 0-3 = multiple    / 4-7 = detune

	５		：slot 1 bit 0-4 = attack rate / 6-7 = key scale
	６		：slot 2 bit 0-4 = attack rate / 6-7 = key scale
	７		：slot 3 bit 0-4 = attack rate / 6-7 = key scale
	８		：slot 4 bit 0-4 = attack rate / 6-7 = key scale

	９		：slot 1 bit 0-4 = decay rate  / 7 = am on
　　　１０		：slot 2 bit 0-4 = decay rate  / 7 = am on
　　　１１		：slot 3 bit 0-4 = decay rate  / 7 = am on
　　　１２		：slot 4 bit 0-4 = decay rate  / 7 = am on
		
　　　１３		：slot 1 bit 0-4 = sustain rate
　　　１４		：slot 2 bit 0-4 = sustain rate
　　　１５		：slot 3 bit 0-4 = sustain rate
　　　１６		：slot 4 bit 0-4 = sustain rate

　　　１７		：slot 1 bit 0-3 = sustain level / 4-7 rerease rate
　　　１８		：slot 2 bit 0-3 = sustain level / 4-7 rerease rate
　　　１９　		：slot 3 bit 0-3 = sustain level / 4-7 rerease rate
　　　２０		：slot 4 bit 0-3 = sustain level / 4-7 rerease rate

　1-G: Vibrato and envelope data

	　M5TB13.SRC has an envelope table and a vibrato table. Each table is signed data.
	　The vibrato table is used in both FM and PSG.
	　The envelope is matched against the VOLM data each time the sound routine is called, valid for PSG only.
	
		The format is

			LAVEL:
				DB <data>,<data>, .... ,<table command>
		
	　The following table commands are also available.


	***** vibrato, envelope table command *****


	TBREPT | Return to the top of the table.
			Format: TBREPT

	TBSTAY | To keep the current output value.
			Format: TBSTAY

	TBBAK | Repeats from the beginning of the table back to the specified number of data.
			Format: TBBAK,<DATA>

	TBEND | For envelopes, stops the sound.
				Same as TBSTAY for vibrato.
			Format: TBEND

	TBADD | FREQ MULTIPLE DATA
							　　　　 (VIBRATE ONLY)
			Format: TBADD,<DATA>

	　






2: Data format details


　2-B: Pitch data
	　In the note mode, note length, notes and commands are assigned as follows.
		Note length   1-7FH
		Note               80H-DFH
		Command     E0H-FFH


	Normal note mode
	　Format: Note data, note length data, note data, note length data

	　The note data has 8 octaves from C0 to BS7.

	　C0 is set to 854H in PSG frequency data and 284H in FM.

	　Transposition can be done with BIAS data, where 1 is a semitone. In BIAS data, 1 corresponds to a semitone, so 0FFH will produce a semitone lower.


	Note Portamento Mode
	  Format: note data, signed change, note length data
	    Example: DB C3,-1,6

	　In this mode, starting from the initial pitch, each time the sound routine is called, the amount of change is added to the previous pitch and the operation is repeated for the note length.

	　If the table data is
		　DB C0,010H,4
	the actual data is
		PSG FREQ FM FREQ
          1: 854H 284H
	  2: 864H 294H
	  3: 874H 2A4H
	  4: 884H 2B4H
        The higher the data, the lower the sound.

	　In PSG, the higher the data, the lower the pitch, and in FM, the opposite is true.

	Direct Frequency Normal Mode
	　Format: FREQ data HIGH, LOW, tone length

	　The table FREQ data and the BIAS value are added together and set to the chip as the pitch.

	　If BIAS is 0 and table data is
		　DB 03H,050H,4
	　then the actual output data is
	          1: 350H
		  2: 350H
		  3: 350H
		  4: 350H
	　The actual output data will be


	Direct Frequency Portamento Mode
	　Format: FREQ data HIGH, LOW, amount of change, sound length data

	　In this mode, starting from the initial pitch (table data + BIAS value), each time the sound routine is called, the amount of change is added to the previous pitch and the output operation is repeated for the length of the note.

	　If the table data is
		　DB 2,0F0H,010H,4
	the actual data is
			 FREQ
        	  1: 2F0H
		  2: 300H
		  3: 310H
		  4: 320H
       and so on.


　2-C: Rhythm data

	　The FM and PSG data are recalled sequentially from the table in M5FMDR13.SRC.
	(DRTB for FM and PSG_DRTB for PSG).

	　The DRUM data format is similar to that of S.E. (except for the header).

	　The DRUM bit assignments are as follows

			7 6 5 4 3 2 1 0
			| |     |--------- FM RYTHM NO.
			| |
			| |----------------PSG RYTHM NO.
			|------------------RYTHM FLAG

	　Therefore, there can be 15 FM rhythms (1-15) and 7 PSG rhythms (1-7).
	　
	　When the table data is 92H, PSG NO. is 1, so we scan PSG_DRTB and get the address of P_HI. With this data, the PSG rhythm is sounded. When the FM NO. is 2, scan DRTB to get the address HTOM. This data gives the FM rhythm. In the case of HTOM, NO. is 0, so the tone of VSNR is set. In the case of HTOM, NO. is 0, so the VSNR tone is set.

	　If you want to add or change the PSG or FM rhythms, add or change the DRTB, DRVTB, PSG_DRTB and table data.

	DRVTB::
		DW	VSNR,VBD,VHHT,VSNR2,VSDSTB,VHIHTTB
	PSG_DRTB:
		DW	P_HI,P_OHI

　2-D: Command data details

	LRPAN L/R setting (FM)
			Format: LRPAN,<L/R DATA>
			　Example: DB LRPAN,LSET		  cf. LRSET equ C0h
							      LSET equ 80h
							      RSET equ 40h
		　Sets the L/R of the FM chip, C0H for L/R, 80H for L and 40H for R.


	FDT detuning (FM/PSG)
	　		Format: FDT,<SIGN DATA>
			  Example: DB FDT,1

		　The pitch value given by NOTE or direct FREQUENCY is added up with the signed data set by the FDT command and set to the chip.

			
	SET_TFLG Timing flag SET (FM/PSG)
			Format: SET_TFLG,<DATA>
			  Example: DB SET_TFLG,80H

		　This sets the data value to T_FLG (0A01C22H). This is used to synchronize the song and character.


	CMTREND Total Release OFF (FM)
			Format: CMTREND
			  Example: DB CMTREND

		　Turns off the total level and release and exits PLAY.


	AUTOPAN AUTOPAN setting (FM)
			Format: AUTOPAN,<PAN NO.>,<PAN TABLE NO.>,
			       <POINT START NO.>,<POINT LIMIT>,<LENGTH DATA>
			  Example: DB AUTOPAN,1,2,1,3,3

		　When the AUTOPAN command is set, the L/R setting is made according to the PAN setting table in <PRJ>CNT.SRC.

		　There are three modes of AUTOPAN, which are set by PAN NO.

	　　Mode 1: (PAN NO. = 1)
	
		　First of all, scan pan_addr_tb of <PRJ>CNT.SRC according to <PAN TABLE NO. > If NO. is 0, L/R setting is done according to the table of pan_0_data, and if NO. is 1, the reading start position of the table is determined by <POINT START NO.> Then, after counting down by <LENGTH DATA>, the next table data is read and set. The counting unit in this mode is the number of sounds. Therefore, after the <LENGTH DATA> is output, the next L/R is set. The data connected by the TIE command is counted as one note. After the table data has been read from the top of the table by <POINT LIMIT> bytes, the data is read again from the top of the table. At this time, <POINT START NO.> is ignored.

		　The following is an example.
		　If the table data is
			DB	AUTOPAN,1,2,1,3,2
			DB	C3,L8,D3,E3,F3,G3,A3,B3
		When the value of <PAN TABLE NO. > is 2, the data is read from the table PAN_2_DATA.
			pan_2_data::
				db	0c0h
			pan_3_data::
				db	040h
				db	080h
				db	0c0h
		　Also, because <POINT START NO.> is 1, 40H, one byte behind the table TOP, is set, and the sound of C3 is emitted from R. Because <LENGTH DATA> is 2, the two sounds up to the sound of D3 are emitted from R. The sounds of E3 and F3 are set to 80H, the second byte from the table TOP, and are emitted from L. ＜Since <LENGTH DATA> is 2, two notes up to the note of D3 come out from R. The notes of E3 and F3 are set to 80H, the second byte from table TOP, and come out from L. The notes of G3 and A3 are set to C0H, the third byte from table TOP, and come out from L/R. Now that the data up to the table TOP + <POINT LIMIT> has been set, the next data reading returns to the table TOP and C0H is set.

	　　Mode 2: (PAN NO. = 2)
		　The difference between Mode 2 and Mode 1 is
		(A) The counting unit is the number of CALLs in the sound routine (by interrupt or FM timer).
		(B) After <POINT LIMIT>, the output is not looped and remains unchanged.
		(C) The sound routine is initialised when the next sound is played.
		
		　Here is an example. Assume that the sound routine is CALLed in the interrupt.
		
		　If the table data is
			DB	AUTOPAN,2,3,0,3,2
			DB	C3,L8,D3,E3,F3,G3,A3,B3
