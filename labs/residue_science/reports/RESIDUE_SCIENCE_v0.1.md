# Residue Science v0.1

**Status:** provisional lab report  
**Lab:** `labs/residue_science/`  
**Role:** first-pass operating model for residue, reconstruction, missing evidence, and false-continuity pressure  
**Canon status:** evidence pressure only; not promoted canon

Best compression:

> **Residue is what lets a system remain intelligible after the original moment is gone.**

Second compression:

> **A good archive preserves reconstruction, not clutter.**

Third compression:

> **The absence of residue is evidence of uncertainty, not permission to invent continuity.**

---

## 1. Why This Report Exists

The current Helix architecture already uses residue-like logic across domains: revision history, evidence doors, source lineage, archived exports, playthrough memory, body-map sensation, loop return, continuity, irreversible commitment, and report pressure.

This report names that operation as a bounded lab method.

Residue Science is not being promoted as a domain. It is a methodological lens for testing whether transformations remain reconstructible.

The immediate value is practical:

```text
claim exists now
→ what evidence created it?
→ what changed during compression?
→ what traces survived?
→ what gaps remain?
→ what confidence is justified?
```

---

## 2. Core Distinction

Residue Science separates three things that often get confused.

### 2.1 Residue

A residue is a trace that remains after a crossing.

Examples:

- commit diff
- deleted path reference
- source citation
- report finding
- session harvest
- operator correction
- compact export
- body soreness after exertion
- game save / base / completion record
- music scrobble / local loved track
- wiki revision history

Residue is useful when it allows reconstruction.

### 2.2 Sediment

Sediment is accumulated material that may or may not help reconstruction.

Examples:

- stale drafts
- duplicate reports
- old planning files
- unlinked dumps
- raw exports with no manifest
- screenshots without context
- obsolete schema fragments

Sediment is not automatically bad. It becomes bad when it blocks the active map or creates search fog.

### 2.3 False Continuity

False continuity is a smooth story invented from insufficient residue.

Typical form:

```text
current clean structure
→ assumed intentional lineage
→ unsupported reconstruction
```

Residue Science exists mostly to stop this.

---

## 3. Working Chain

The lab uses this chain:

```text
prior state
→ crossing / transformation
→ residue
→ decay or preservation
→ reconstruction
→ confidence tier
→ action
```

A reconstruction is only as strong as its trace quality.

If the residue is thin, confidence should drop instead of being patched with aesthetic coherence.

---

## 4. Relation to Existing Helix Concepts

### 4.1 Compression

Residue Science protects against destructive compression.

Good compression preserves enough doors to reopen evidence.

Bad compression makes the prose cleaner while cutting the path back to why the claim exists.

### 4.2 Reports

Reports are intermediate residue.

They are valuable because they preserve pressure, uncertainty, and interpretation. But they do not become canon unless promoted through review.

### 4.3 Domains

Domain files are compressed residue plus local interpretation.

They should preserve examples because examples are evidence doors.

### 4.4 Archive

Archive is not canon.

Archive is heavy provenance that may allow reconstruction if compact layers become insufficient.

### 4.5 AOC / DCP / LIP / EIP

Residue Science has a strong methodological bridge to the AOC cluster:

```text
LIP = pre-crossing insufficiency
DCP = crossing / commitment
EIP = post-crossing residue
Residue Science = reconstructability audit after the crossing
```

This bridge should not be overpromoted. It is useful because it gives the theory an audit surface, not because similar language proves the theory.

## 5. Residue Classes

| Class | Definition | Example | Risk |
|---|---|---|---|
| Direct residue | Trace closest to the event | commit diff, operator correction, raw export | may still lack motive/context |
| Intermediate residue | Processed evidence | report, session harvest, normalized dataset | may be mistaken for canon |
| Compressed residue | High-level summary | domain compression, DISSONANCE line | may lose evidence doors |
| Negative residue | Expected trace missing | absent source, missing report | may be overread |
| Decayed residue | Trace exists but weakened | stale export, broken link, vague note | may create false confidence |
| False residue | Looks evidential but does not support claim | unrelated source, coincidental wording | may anchor wrong story |
| Reconstructive residue | Enough trace to rebuild lineage | report + source + correction | still requires confidence tier |

## 6. Manual Audit Procedure

A manual residue audit should answer seven questions:

```text
1. What object is being reconstructed?
2. What is the current state?
3. What prior state is claimed or implied?
4. What crossing changed it?
5. What residue survived?
6. What is missing or decayed?
7. What confidence and action are justified?
```

Then classify the outcome:

```text
CLAIM_STABLE
CLAIM_PARTIAL
CLAIM_UNTRACED
CLAIM_CONFLICTED
CLAIM_OVERCOMPRESSED
CLAIM_DEMOTE_TO_ANOMALY
```

## 7. First Audit Targets

### Target A — DISSONANCE compression line

Question:

> Can a high-level compression line be traced back to concrete domain examples?

Pass condition:

```text
global line
→ domain section
→ examples / corrections / data
→ confidence tier
```

Failure condition:

```text
global line
→ sounds right
→ no traceable evidence doors
```

### Target B — Deleted or renamed file path

Question:

> Was the transformation useful consolidation or load-bearing residue loss?

Pass condition:

```text
old path
→ commit / report / replacement path
→ merged content preserved
```

Failure condition:

```text
old path gone
→ no replacement
→ no anomaly
→ current agents cannot reconstruct why
```

### Target C — AOC transition wording

Question:

> Does the LIP/DCP/EIP crossing language produce auditable evidence expectations?

Pass condition:

```text
pre-crossing insufficiency
→ crossing / commitment event
→ post-crossing residue
→ falsifier or demotion rule
```

Failure condition:

```text
language is elegant
→ no observable trace class
→ no test path
```

### Target D — Music or games data claim

Question:

> Can a domain claim that relies on exports still find its source layer?

Pass condition:

```text
raw / compact export
→ domain interpretation
→ global pattern candidate
```

Failure condition:

```text
claim survives
→ export date/source absent
→ no way to evaluate drift
```

## 8. Example Residue Card: DCP / EIP Wording

**Object:** transition-language bridge between DCP and EIP.

**Current state:** DCP is treated as crossing / commitment; EIP is treated as post-crossing residue.

**Prior state:** older theory work used commitment, irreversibility, and residue language less cleanly across multiple documents.

**Crossing / transformation:** AOC consolidation compressed LIP/DCP/EIP/OCH into a single hierarchy.

**Surviving residue:** active AOC README, theory file, reports, claim ledger candidates.

**Missing / decayed evidence:** exact first wording lineage may require chat/session harvest or commit history.

**Confidence:** Tier 2 unless direct report/commit line confirms the wording origin.

**False-continuity risk:** assuming the current clean hierarchy existed from the beginning.

**Recommended action:** preserve current bridge, but require specific report citation before using it as evidence rather than orientation.

## 9. Failure Modes

### 9.1 Archive hoarding

Preserving every trace can make reconstruction worse.

Countermeasure:

```text
preserve load-bearing residue
compress reviewed residue
remove stale husks after useful content is merged
```

### 9.2 Narrative smoothing

Agents may invent clean continuity because a current system looks orderly.

Countermeasure:

```text
require prior-state evidence before claiming lineage
```

### 9.3 Gap paranoia

Missing evidence may be treated as a meaningful hidden pattern.

Countermeasure:

```text
gap = uncertainty unless supported
```

### 9.4 Report canonization

A report may be treated as promoted truth.

Countermeasure:

```text
reports are pressure artifacts until reviewed
```

### 9.5 Metaphor inflation

Residue language may spread into poetic explanation without audit value.

Countermeasure:

```text
every residue claim must name an actual trace
```

---

## 10. What Success Looks Like

Residue Science succeeds if it makes future work more honest.

Success examples:

- a claim gets strengthened because its lineage is traceable,
- a claim gets demoted because it cannot find its evidence,
- an anomaly is created instead of silently reconciling contradiction,
- a stale file is deleted after useful content is merged,
- a deleted path is documented as non-load-bearing,
- an agent stops itself from inventing a smooth history,
- a domain compression keeps enough examples to reopen the room.

It fails if it becomes another pile of reports about how everything is residue.
