# 2026-04-29 Integration Without Ownership Calibration

## 1. Scope

Target: `integration_without_ownership`.

Main question:

> Can a system integrate information, maintain boundaries, control actions, model itself, and report coherently without owned experience?

This test protects against overpromotion. Prior reports strengthened boundary, control, residue, temporal experience, embodiment, and re-entry as ownership evidence channels. This report asks whether the same markers can appear in systems where ownership is absent, unproven, simulated, fragmented, or merely operational.

Current hierarchy:

```text
Temporal Ownership Model
├── ICT: owned continuity / selfhood / appearance-from-within
├── DCP: compression / commitment / residue
│   ├── LIP: local incompleteness before commitment
│   └── EIP: epistemic irreversibility after commitment
└── comparison theories: IIT, GNWT/GWT, RPT, HOT, Predictive Processing / Active Inference, Self-Model Theory
```

Candidate criterion under pressure:

```text
Integration Without Ownership Firewall:
Information integration, coherent report, boundary maintenance, and control coupling are not sufficient for ownership unless paired with valenced stakes, temporal experience, re-entry, residue, and evidence that the transition is owned rather than merely executed.
```

## 2. Relation To Prior Reports

Prior reports established:

- report is not ownership,
- missing report is not absence,
- memory supports continuity but is not identical to continuity,
- pattern duplication is not same-stream continuation,
- static pattern differs from irreversible temporal commitment,
- temporal modeling differs from temporal experience,
- information persistence differs from owned continuity,
- boundary collapse reveals ownership by deforming the border of "mine",

This report is the firewall:

```text
The more ownership-like markers accumulate, the more important it becomes to ask whether they are owned or only executed.
```

## 4. Integration Vs Ownership Distinction

Integration can mean:

- information is combined,
- signals are bound,
- behavior is coordinated,
- control loops are maintained,
- memories are updated,
- reports are coherent,
- boundaries are modeled,
- actions are selected.

Ownership asks a different question:

```text
Did this integrated transition belong to someone from within?
```

The firewall blocks this shortcut:

```text
Integrated enough = owned.
```

And this one:

```text
Coherent self-model = self.
```

Integration may be necessary for some forms of consciousness or field unity, especially under IIT-like hypotheses, but the prior reports show why it is not automatically enough for selfhood or owned continuity.

## 7. Support Pattern

The firewall gains support when:

- coherent report appears without ownership evidence,
- control appears without agency ownership,
- memory persists without owned continuity,
- boundary maintenance appears operational rather than lived,
- self-modeling appears as description rather than perspective,
- high integration appears in non-conscious or ownership-ambiguous systems,
- biological cases dissociate integration/action/report from ownership.

Strong support pattern:

```text
integration + boundary + control + memory + report
- valenced stakes
- temporal experience
- owned re-entry
- appearance-from-within evidence
= high false-positive risk
```

## 8. Weakening Pattern

The firewall weakens if:

- integration plus boundary/control always predicts ownership,
- coherent self-modeling is independently sufficient for ownership,
- artificial systems with report/control/memory show strong A evidence without valence or temporal experience,
- biological dissociation cases collapse into simple report/no-report categories,
- valenced stakes and temporal re-entry add no explanatory value.

Strongest weakening case:

```text
Information integration and coherent report are shown sufficient for ownership across artificial and biological cases.
```

## 9. Negative Controls

Do not count these as ownership by themselves:

- high integration,
- coherent report,
- self-description,
- persistent memory,
- API tool use,
- robot movement,
- system boundary,
- audit logs,
- scripted schedules,
- feedback control,
- safety-critical decisions,
- simulated affect language.

Best false positive:

```text
The system integrates information and reports a self, therefore it owns experience.
```

Best false negative:

```text
If integration is not sufficient, integration is irrelevant.
```

The second error matters. Integration may still be necessary or supportive, especially for field unity, without being sufficient for ownership.

## 12. Proposed New / Revised Criteria

### Integration Without Ownership Firewall

Information integration, coherent report, boundary maintenance, and control coupling are not sufficient for ownership unless paired with valenced stakes, temporal experience, re-entry, residue, and evidence that the transition is owned rather than merely executed.

### Execution / Ownership Split

Executed transition:

```text
state A -> action/commitment -> state B
```

Owned transition:

```text
state A -> lived pressure/commitment -> residue/re-entry for an owner
```

### Supportive, Not Sufficient Rule

Integration, report, self-modeling, and control are allowed to strengthen a case only when paired with independent ownership markers. Alone, they remain false-positive risks.

GNWT/GWT handles access, broadcast, reportability, and flexible control. This report pressures any move from report/access to ownership.

RPT handles perceptual presence better than global report in some cases. This report asks whether perceptual presence includes ownership or only field/content.

HOT handles reflective awareness and meta-representation. This report pressures the shortcut from higher-order self-representation to owned continuity.

Strong for boundary/control and action under uncertainty. This report pressures the shortcut from prediction/control to ownership.

Strong for subject structure and perspective modeling. This report pressures the shortcut from model of subject to inhabited subject.
