# 2026-04-29 Two-Mode Compression Calibration

## 1. Scope

This report pressure-tests whether the proposed compression split is structural or merely descriptive.

Candidate split:

```text
Curatorial Compression:
  target shape known or strongly implied
  repairs an object toward its proper/usable form

Investigative Compression:
  target shape unknown or underdetermined
  narrows possibility space by killing false positives
```

Main question:

> Is this a real structural split in the operator's cognition and Helix method, or only a convenient label pair?

This report does not promote the split to canon. It asks what the split predicts, where it blurs, and what would falsify it.

## 2. Candidate Definitions

### Curatorial Compression

A compression is curatorial when the system already has an available standard, target grammar, or expected final shape, and the work removes noise while preserving/recovering source lineage and functional identity.

Core question:

> What should remain so this object expresses its proper form?

Best compression:

> Curatorial compression makes an object more itself.

### Investigative Compression

A compression is investigative when the target is not yet known and the work reduces possibility space by eliminating false positives, clarifying contrast classes, and making the remaining question more difficult to fake.

Core question:

> What can be ruled out so the real question remains?

Best compression:

> Investigative compression makes a mystery harder to fake.

## 4. Curatorial Compression Analysis

Curatorial compression is strongest where the operator can already sense the object's proper behavior.

Wikipedia is the cleanest case. A page has public-facing constraints: lead/body alignment, source lineage, MOS, neutral weight, article anatomy, and future-reader wayfinding. Trimming is not anti-depth when it removes fake depth and lets the subject behave more like itself.

Helix refactors show the same mode. Once the root grammar is known, folders that repeat names, preserve stale lifecycle buckets, or keep dead scaffolding become clutter. The target is not "less stuff"; the target is a browsable body whose path explains the thing before a README has to defend it.

Food order simplification is a small-bodied version: reduce friction around a known comfort grammar. The goal is not discovery of a grand food theory; it is making the meal pass the edible, low-friction, stable-base gate.

Strongest support:

```text
Curatorial cases feel like repair because a target grammar already exists.
```

Overcompression risk:

```text
Known target -> deletion confidence -> load-bearing examples disappear.
```

## 5. Investigative Compression Analysis

Investigative compression is strongest where the target is hidden by attractive false substitutes.

The Temporal Ownership reports are the cleanest cases. Each report asks what common substitute fails:

- report is not ownership,
- lack of report is not absence,
- memory is not continuity,
- pattern duplication is not singular continuation,
- temporal modeling is not temporal experience,
- information persistence is not owner persistence,
- integration is not ownership.

This mode does not repair a known object toward a standard. It tightens the question until weaker explanations fail.

Music response-axis discovery is an earlier aesthetic version. The target axes were not known at the start. Genre labels, play counts, and audio features were pressure-tested until "floor/body," "air/timbre," "constraint," and "return" became plausible response axes.

Strongest support:

```text
Investigative cases feel like narrowing because the target is not available yet.
```

Overcompression risk:

```text
Eliminating alternatives can feel like progress even when no positive criterion has been found.
```

## 6. Mixed Cases

Some high-value Helix work uses both modes.

### DISSONANCE.md Regeneration

Curatorial: preserve the person-pattern, evidence doors, domain chambers, and portable readability.

Investigative: identify which compressions are false, which examples are load-bearing, and which global candidates are overpromoted.

Output: improved object plus sharper questions.

### Domain File Patching

Curatorial when the domain has known local grammar.

Investigative when the patch is discovering a new gate, false positive, or negative control.

### Trails Database / Atlas

Curatorial: source ingestion, spoiler bands, database hygiene, public/world-memory structure.

Investigative: hidden institution/world-memory reconstruction, arc-level pattern inference, continuity questions.

### Helix Architecture Refactor

Curatorial once the root grammar is known.

Investigative while discovering which folders are real rooms versus stale scaffolding.

### Dataset Triage

Curatorial: registry files, metadata, local paths, notes.

Investigative: choosing which dataset can pressure which false substitute.

### Wiki Good Article Work

Curatorial: lead/body/source/interface repair.

Investigative: deciding what article-specific structure is load-bearing and what is cruft.

## 7. Support Pattern

The split is supported when cases differ along these axes:

- known target vs unknown target,
- repair toward standard vs elimination of false substitutes,
- source lineage restoration vs contrast-class sharpening,
- improved object vs sharper question,
- public usability vs mystery resistance,
- trim-to-form vs narrow-to-A.

The split predicts that the same operator can both:

```text
clean a page because the shape is known
```

and:

```text
pressure consciousness theory because the shape is not known
```

without contradiction.

## 8. Weakening Pattern

The split weakens if:

- all curatorial cases require open-ended discovery,
- all investigative cases secretly rely on known target standards,
- mixed cases cannot be assigned even locally,
- the labels do not change agent behavior,
- the split only renames "editing" and "research,"
- compression outcomes cannot be predicted differently by mode.

Strongest counterexample:

```text
Wiki Good Article work can be both highly standardized and highly investigative.
```

This does not falsify the split; it shows modes can stack in phases.

## 9. Negative Controls

False positives:

- calling every cleanup curatorial compression,
- calling every research question investigative compression,
- treating the split as proof of DCP,
- treating the operator's cognitive style as universal law,
- treating removal/deletion as automatically good compression,
- treating two labels as a complete theory of cognition.

Negative control:

```text
A random tidy-up that removes visible clutter without source lineage, target grammar, or future usability is not curatorial compression. It is just cleaning.
```

Another:

```text
A vague brainstorm that produces more possibilities without killing false substitutes is not investigative compression. It is expansion.
```

## 10. Relation To DCP / LIP / EIP

DCP should not collapse into one mode.

Candidate refinement:

```text
DCP-C / Curatorial:
  compression toward a known or strongly implied target form

DCP-I / Investigative:
  compression through elimination of false positives and narrowing of possibility space
```

LIP attaches differently:

- curatorial LIP: local clutter may hide the object's proper form;
- investigative LIP: local evidence is insufficient because the target is underdetermined.

EIP attaches differently:

- curatorial EIP: every repair leaves revision residue and changes future maintenance;
- investigative EIP: every eliminated false positive changes what can be believed next.

DCP remains the commitment/compression crossing:

```text
LIP detects insufficiency
DCP compresses toward form or through elimination
EIP tracks residue
ICT asks whether any crossing was owned
```

## 11. Relation To TOM / ICT

TOM can use the split as a method-level refinement: it clarifies how reports pressure claims and how workspace repair differs from consciousness inquiry.

ICT should not treat the split as evidence for appearance-from-within. The split is about the operator's cognitive method and DCP's possible operating modes, not proof of consciousness.

## 13. What This Does Not Prove

This report does not prove:

- DCP is true,
- the operator's cognition is universal,
- deletion is automatically good,
- every research process is investigative compression,
- every cleanup is curatorial compression,
- compression is always the right move,
- the split solves consciousness.

## 14. Falsifiers

The split should be rejected or revised if:

- mode classification does not improve predictions,
- agents make worse edits after using it,
- mixed cases dominate so completely that the distinction adds no information,
- "known target" and "unknown target" fail to separate actual workflows,
- the same evidence supports both labels with no phase distinction,
- the split becomes a flattering self-description instead of a falsifiable operating handle.

## 15. Compression

> Curatorial compression makes an object more itself.

Second:

> Investigative compression makes a mystery harder to fake.

Third:

> One trims the lawn; the other narrows the cave.

Fourth:

> The first repairs a known form; the second pressures an unknown one.

## 16. Recommendation

Recommendation:

```text
keep as DCP subprimitive
```

Reason:

The split has enough predictive clarity to guide agents and theory reports, but not enough independent pressure to become a core TOM primitive. It should live as a candidate DCP refinement and operating rule:

```text
DCP-C = compression toward known form
DCP-I = compression through false-positive elimination
```

Next review trigger:

Promote only if future reports show the split repeatedly predicts overcompression risk, output type, evidence-handling behavior, and agent success better than the older single "compression-native cognition" label.
