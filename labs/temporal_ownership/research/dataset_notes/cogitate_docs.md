# COGITATE Docs-Only Intake

## Source

- **Project:** COGITATE adversarial collaboration / IIT vs GNWT
- **Dataset documentation:** https://cogitate-consortium.github.io/cogitate-data/
- **Nature article:** https://www.nature.com/articles/s41586-025-08888-1

## Scope

Docs-only note. No raw fMRI, M-EEG, iEEG/ECoG, behavior, or eye-tracking files were downloaded.

## Dataset Facts

Data-release documentation describes 256 total subjects:

- 118 fMRI
- 100 M-EEG
- 38 iEEG/ECoG

The Nature article reports final analyzed samples after exclusions separately, so both sets of numbers should be kept distinct.

Design variables relevant to TOM:

- task relevance,
- stimulus duration,
- stimulus category,
- orientation,
- content maintenance,
- access/report conditions.

## First TOM Use

Use COGITATE as a theory-comparison benchmark before raw analysis:

1. Build a variable map.
2. Separate access/report variables from binding/integration variables.
3. Identify where temporal maintenance enters the design.
4. Ask which observations would pressure IIT, GNWT/GWT, RPT, and TOM differently.

## Relevance To TOM / ICT / DCP / LIP / EIP

- **TOM:** helps separate access, binding, temporal maintenance, and report.
- **ICT:** relevant to appearance/access distinction, but not directly a selfhood dataset.
- **DCP:** stimulus duration and task commitment may become transition/maintenance proxies.
- **LIP:** local neural signatures underdetermine ownership and appearance-from-within.
- **EIP:** trial structure and report leave residue, but residue is not ownership.

## Limitations

- COGITATE tests IIT/GNWT predictions, not TOM directly.
- The first TOM step should be conceptual/variable mapping.
- Raw data should wait until a specific question exists.
- Do not use COGITATE as proof for or against ICT.
