## Phase 1a: Small Dataset Triage

Tasks:

- verify dataset links and access paths,
- collect only metadata/docs first,
- create `dataset_cards/*.md` or `dataset_notes/*.md` if useful,
- choose one tiny pilot.

Tiny pilot options:

1. DREAM metadata subset.
2. ds004902 one-subject EEG metadata/pilot.
3. COGITATE design/variable map only.

Do not download large raw data.

## Phase 1b: Small-First Dataset Intake

Priority:

1. DREAM metadata package.
2. DREAM smallest open dataset candidate after registry inspection.
3. ds004902 one-subject pilot.
4. ds006623 derivative-only pilot.
5. COGITATE design/variable map.

Rules:

- keep intake to registry files, metadata, documentation, and tiny derivatives,
- do not download full EEG/MEG/fMRI datasets by default,
- write dataset notes before raw analysis,
- treat datasets as pressure tools, not proof machines,
- promote no claims from intake alone.

## Phase 2: Dataset-Specific Analysis Plans

Create dataset-specific plans without downloading data.

Each plan should state:

- target claims,
- variables/proxies,
- small-first access path,
- likely false positives,
- likely false negatives,
- what would strengthen,
- what would weaken,
- what would remain open.

## Phase 3: Minimal Reproducible Pilot

Run one minimal pilot only after Phase 1a chooses a tiny target.

Priority:

1. DREAM metadata subset if accessible.
2. ds004902 one-subject metadata/pilot.
3. COGITATE variable map only.

## Phase 4: Claim Pressure Review

Map results to ledgers only as pressure:

- strengthen,
- weaken,
- split,
- demote,
- leave open.
