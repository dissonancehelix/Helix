# Temporal Ownership

Temporal Ownership is the active cross-domain research room for time, ownership, commitment, access, binding, report, selfhood, and continuity.

## Active Model

Temporal Ownership Model is the umbrella / integration model.

- `THEORY.md` is the root orientation.
- `core/` is the active conceptual harness for ICT/TOM tests.
- `research/` contains datasets, literature, small-first data triage, and analysis lanes.
- Reports are evidence, not canon.
- Claims are promoted only through review.
- Obsolete or superseded docs are deleted after useful content is merged.

Working compression:

> LIP is before the crossing, DCP is the crossing, EIP is the residue after the crossing, and ICT asks whether the crossing was owned.

External theories are comparison pressure, not replacements:

- IIT: binding / intrinsic integration / field candidate.
- GNWT/GWT: access / broadcast / reportability.
- RPT: recurrent perceptual presence.
- HOT: reflective awareness / meta-representation.
- Predictive Processing / Active Inference: boundary, prediction, control coupling.
- Self-Model Theory: subject structure, ownership model, perspective.

## Navigation Rule

Use the smallest active source:

- Use `THEORY.md` for orientation.
- Use `core/` for conceptual tests, primitives, claims, falsifiers, and reports.
- Use `research/` for datasets, literature, small-first triage, and analysis lanes.

Do not recreate root-level duplicate ledgers, stale runners, or broad planning files.
