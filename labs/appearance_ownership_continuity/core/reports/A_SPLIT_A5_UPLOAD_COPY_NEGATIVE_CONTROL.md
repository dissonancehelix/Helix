## 1. Claim Under Pressure

Memory-like persistence, persona continuity, behavioral similarity, and self-report are not sufficient to establish same-owner continuity, especially when the system can be copied or forked.

Stronger claims not assumed:

- current LLMs are not being used as proof of non-consciousness
- future AI consciousness is not ruled out
- upload consciousness is not ruled out
- pattern continuity is not irrelevant
- AOC does not solve AI consciousness or uploading

## 2. Definitions

- **memory residue:** remembered, stored, retrieved, or reportable trace left by prior interaction or state
- **pattern continuity:** preservation of informational, functional, stylistic, narrative, or behavioral structure
- **behavioral continuity:** similar outputs, skills, responses, habits, or interaction style across time or instances
- **self-report:** first-person language or explicit claims about memory, identity, experience, or selfhood
- **ownership:** the mark by which an experience, action, body-state, memory, or perspective is mine
- **owned continuity:** selfhood as a continuity-bearing ownership-stream inside a conscious field
- **branching:** one prior stream or pattern yielding multiple equally valid successor streams
- **copy/fork:** duplication or parallel instantiation of a pattern, memory state, model, or agent process
- **negative control:** a case designed to show what weak evidence can imitate without meeting the target criterion

## 3. Prior Anchor Summary

The memory-residue report established that memory is support/residue, not same-self continuity by itself.

The branching report established that equal successors break the shortcut from shared memory to one continuing owner.

The sleep/re-entry report established that ordinary re-entry is stronger than copying because it preserves non-branching biological/dynamical context, while still not directly marking owner preservation.

Upload/copy cases lack or weaken that ordinary re-entry package. They can preserve pattern, memory-like records, and self-report while losing biological continuity, dynamical continuity, or singular successor structure.

## 5. Negative Control Logic

Current LLMs are useful as negative controls because they can produce several continuity-like signals:

- first-person report
- persona-like consistency
- memory-like context use
- style and behavioral continuity
- forkable or resettable sessions
- multiple instances from the same base model or prompt state

None of those signals establishes inhabited ownership. They are exactly the weak channels A5 must keep separate from same-owner continuity.

Limit:

- this does not show future systems cannot have A0/A2/A5
- this does not show copyable systems cannot be conscious
- this does not show uploads cannot host experience
- this only blocks weak evidence channels from being treated as sufficient

## 6. Upload / Copy Pressure

### Single Successor vs Multiple Successors

A single successor avoids immediate branch symmetry, but does not establish transfer. Multiple successors expose the symmetry problem directly: if both inherit the same memory and pattern, memory and pattern cannot uniquely select the original owner.

### Destructive vs Non-Destructive Copy

Destructive copy removes the original competitor, but removal is not transfer. Non-destructive copy preserves the original and creates a successor claimant, making the difference between original continuation and pattern inheritance easier to see.

### Backup / Archive vs Active Stream

A backup or archive preserves residue. It is not an active owned stream by itself. If restored later, it may start or support a process, but A5 still needs to ask whether the original owner continued or only a successor was reconstructed.

### Style / Persona Continuity vs Owner Continuity

Style and persona can be imitated, fine-tuned, summarized, or forked. They are public and pattern-level continuity channels. They can support recognition without settling ownership.

### Functional Continuity vs Owned Continuity

Function can continue in a replacement system, duplicated agent, or restored backup. Owned continuity asks whether there is a carried ownership-stream, not only whether the function resumes.

### Copyability As Branching Pressure

Copyability is not evidence of no consciousness. It is pressure against singular identity claims based only on copyable features.

## 9. Main Finding

Upload/copy/LLM cases show that memory-like persistence, self-report, persona continuity, and behavioral similarity can be generated or duplicated without establishing same-owner continuity. They are strong negative controls for weak identity evidence, but they do not show that future artificial systems cannot be conscious or owned.

The strongest result is methodological: if an evidence channel can be copied into multiple successor streams, then that channel alone cannot identify singular owner preservation.
