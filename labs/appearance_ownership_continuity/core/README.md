# AOC Core Harness

**Status:** minimal theory-testing scaffold  
**Parent theory:** `labs/appearance_ownership_continuity/THEORY.md`  
**Scope:** claim, primitive, falsifier, comparison, and test registry for the AOC Framework  
**Not role:** replacement theory, proof of consciousness, metaphysics file, religion file, or empirical certainty machine

## Naming

Current umbrella:

> Appearance–Ownership–Continuity Framework (AOC Framework)

Deprecated working titles:

- Temporal Ownership Model / TOM
- Inhabited Continuity Theory / ICT

Current sub-hypothesis:

> Owned Continuity Hypothesis (OCH)

`core/` contains the active conceptual test harness for the AOC Framework. It is not a separate theory and does not preserve the older `inhabited_continuity` folder name. OCH remains a named sub-hypothesis inside AOC; the folder is named for function, not lineage.

This harness keeps Owned Continuity Hypothesis reviewable by separating claims, primitives, tests, comparisons, thought experiments, empirical contacts, and falsifiers.

The parent theory remains `labs/appearance_ownership_continuity/THEORY.md`. This folder does not rewrite it. It is a pressure harness around the current stance:

```text
consciousness = field of appearance
self = owned continuity inside that conscious field
central unresolved gap = A, appearance-from-within
```

## Relation To Worldview

`domains/worldview/WORLDVIEW.md` owns belief/confidence boundaries around religion, souls, rebirth, metaphysics, morals, institutions, and power.

This harness owns only the research question:

> What must be preserved for an inhabited interior to remain the same self through time?

Speculative soul/rebirth language may appear only as continuity-carrier pressure, not as proof or doctrine.

## How To Use

1. Start with `PRIMITIVES.md` to name the moving parts.
2. Use `CLAIM_LEDGER.yaml` to see claim status and dependencies.
3. Use `TEST_REGISTRY.yaml` to select a pressure test.
4. Compare against existing theories in `THEORY_COMPARISON.md`.
5. Record generated analysis under `reports/`.
6. Promote, demote, or revise claims only after a test creates real pressure.

Reports should specify whether they are using:

- **curatorial compression**: repairing toward a known or strongly implied form,
- **investigative compression**: narrowing an underdetermined question by killing false positives,
- **both**: when an object must be repaired while the right boundary is still being discovered.

## Successful Test

A successful test does at least one of these:

- distinguishes consciousness from selfhood,
- distinguishes reportability from ownership,
- separates memory support from identity persistence,
- pressures branching and first-person continuation,
- links a primitive to empirical or clinical evidence,
- weakens an overbroad claim,
- clarifies what A still leaves unexplained.

## Weakening

A claim weakens when a test shows that:

- it conflates reportability, memory, self-modeling, intelligence, or behavior with inhabited selfhood,
- it turns an analogy into evidence,
- it predicts too much,
- a competing theory explains the case with less residue,
- it cannot state what would count against it.

## Promotion And Demotion

Claims move by evidence and pressure:

```text
analogy_only -> speculative -> plausible -> anchored
anchored/plausible -> open/rejected if falsifiers land
```

No claim promotes itself. Reports can recommend movement; review decides.

Best compression:

> **This harness does not prove consciousness; it keeps the theory from dissolving.**
