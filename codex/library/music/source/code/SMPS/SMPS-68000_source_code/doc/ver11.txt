４：ＳＯＮＧ、Ｓ．Ｅのデータ形式

	　下記のもの以外は、Ｍ５　Ｚ８０（Ｖｅｒ１．３）と同じ形式になっていますので、そちらのマニュアルを参考にしてください。

　　※　Ｍ５　Ｚ８０（Ｖｅｒ１．３）との相違点

　　①　ＳＯＮＧ、Ｓ．Ｅのアドレッシング

	　アドレスは、Ｚ８０が１６ｂｉｔであるのに対して６８Ｋは３２ｂｉｔです。このため、Ｚ８０のデータ形式（ＤＷ）にするために、相対アドレスを使用しています。

	１．ヘッダー中のアドレッシングそれぞれのテーブルのトップアドレスから、ＳＯＮＧのトップアドレスを差し引くことで１６ｂｉｔ化します。（プログラムは、ＳＯＮＧのトップアドレスを足すことでテーブルのトップアドレスを算出している。）
	　　また、このマクロを、［ＴＤＷ］とします。

	２．データテーブル中のアドレッシング
	　　データテーブルのジャンプ先は、ジャンプ先のアドレスからジャンプ元のアドレスを差し引くことで１６ｂｉｔ化します。（プログラムは、ジャンプ元のアドレスを足すことでジャンプ先のアドレスを算出している。）
	　　また、このマクロを、［ＪＤＷ］とします。

	　以上をまとめると、
		１．ヘッダー中は、［ＴＤＷ］を用いる。
		２．ＳＯＮＧ、Ｓ．Ｅのテーブル中では、［ＪＤＷ］を用いる。となります。

　　②　ラベル
	　６８Ｋのラベルは必ず偶数アドレスに割り当てられます。このため、奇数アドレスにラベルを指定しようとした場合、ラベルが偶数アドレス化されるために１バイトダミーデータ（００Ｈ）が入ってしまいます。

	例	T810A:
			DB	CN4,L1,DN4
		T810B:
			DB	EN4,GN4

		このような場合、アセンブルされるとデータが次のようになります。

			0B1H,030H,0B3H,000H,0B5H,0B8H

		（０００Ｈという余分なデータが入ってしまう）

	これを避けるためにデータテーブル中のラベルは、

		TAB810	EQU	$	（または　*）

	で与えるようにします。



	例	T810A	EQU	$
			DB	CN4,L1,DN4
		T810B	EQU	$
			DB	EN4,GN4

		このような場合、アセンブルされるとデータが次のようになります。

			0B1H,030H,0B3H,0B5H,0B8H

	※　ただしヘッダーのトップ（Ｓ８１等）は、［ラベル：］で与えてください。（コントロールの都合上［ＥＱＵ　＄］だとアドレスエラーが出ます）

	③　アセンブラ［Ｘ６８Ｋ．ＥＸＥ］の性質上、［ＩＲ８０．ＥＸＥ］とは次のような相違点があります。

	　　・ラベル名、変数名は、大文字と小文字を区別します。
	　　・シンボル出力は、ＰＵＢＬＩＣ疑似命令で行います。ダブルコロン（：：）は使用できません。
	　　・リストファイルへの出力、非出力は、ＬＩＳＴ　ＯＮ／ＯＦＦ　で行います。

	④　ゲームのメインプログラムやメガドライブのハードの都合上、ＦＭタイマーは使用できません。（Ｖブランクのインタラプトで行います。）

	⑤　テーブルビブラート､ワウ、ポルタメントモード、ダイレクトフリケンシーモードはありません。

	以上のことより、Ｍ５　Ｚ８０のファイルをそのまま用いる場合、次の変更が必要です。

	①　ヘッダー中のＤＷをＴＤＷにする。

	　例	DW	TIMB81				TDW	TIMB81,S81
		DB	6,3,TP81,DL81　　　　　→	DB	6,3,TP81,DL81
		DEFW	TAB81D				TDW	TAB81D,S81
		DEFB	0,FA81D				DB	0,FA81D

	②　テーブルデータ中のジャンプ先を表すＤＷをＪＤＷにする。

	　例	DB	CMCALL		　　　→	DB	CMCALL
		DW	T810A				JDW	T810A

	③　ラベル（：や：：）を　ＥＱＵ　＄にする。

	　例	TAB810::		　　　→	TAB810	EQU	$

	④　ラベル名、変数名等の大文字・小文字を統一する。

	　例	Bass1	EQU	3	　　　→　	Bass1	EQU	3
			DB	FEV,BASS1		DB	FEV,Bass1

	⑤　ヘッダーのトップのラベル（Ｓ８１、Ｓ８２等）をＰＵＢＬＩＣ宣言す
	　る。（ＰＵＢＬＩＣ　Ｓ８１等）

	　例	S81::			　　　→		PUBLIC	S81
							S81:
