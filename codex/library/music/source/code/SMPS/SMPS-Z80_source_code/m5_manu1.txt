	３−Ｃ−①）　＜ＰＲＪ＞ＳＮＧ．ＨＥＸ　

	　＜ＰＲＪ＞ＳＮＧ．ＨＥＸ　は、１曲だけをアセンブル、リンクするもの
	です。リンク時にエラーがでますが、構いません。このエラーは、ダミーの
	ラベルを持つことで解決できますが、その場合、そのままでは、全曲をリン
	クして＜ＰＲＪ＞ＢＮＫ．ＨＥＸを作る場合にＭＵＬＴＩ　ＤＥＦＩＮＤ　
	ＥＲＲＯＲが起きるので、マクロのＩＦ文等でとりはずしできるようにして
	おくことが必要です。

	　＜ＰＲＪ＞ＳＮＧ．ＨＥＸのスタートアドレスは、＜ＰＲＪ＞．ＬＮＫと
	いうリンクファイル内で与えています。Ｐ：１４００　がそれです。この場
	合、スタートアドレスが１４００Ｈとしてリンクします。このときＺ８０メ
	モリの１Ｃ０２Ｈを００Ｈ、１Ｃ０３Ｈを１４Ｈとします。（４：データト
	ップアドレスの与え方参照）
		（ＷＯＲＤで　１４００Ｈ。サウンドコントロールは、このアドレ
		スを参照して、曲、Ｓ．Ｅ．、ＥＮＶＥ、ＶＩＢＲ等のデータアド
		レスを得ます。また、エディタも同様です。）
	　また、１Ｃ００Ｈ以降は、プログラムのＷＯＲＫ　ＲＡＭなので、＜ＰＲ
	Ｊ＞ＳＮＧ．ＨＥＸは、１ＢＦＦＨまでに納める必要があります。



	３−Ｃ−②）＜ＰＲＪ＞ＳＥ．ＨＥＸ

	　＜ＰＲＪ＞ＳＥ．ＨＥＸ　は、ＳＥデータをリンクしたものです。スター
	トアドレスは　＜ＰＲＪ＞．ＭＫＦのＬＮＫコマンドで与えています。この
	アドレスは、つぎに述べる”Ｓ．Ｅ．データのスタートアドレス”と同一で
	なくてはなりません。
	
	　Ｓ．Ｅ．データのスタートアドレスは、＜ＰＲＪ＞ＴＢ．ＬＩＢで
		setop_adr	equ	1000h
	として、プログラムに与えます。このアサインは＜ＰＲＪ＞ＴＢ．ＳＲＣの
	ＡＤＤＲＴＢ：：６ＢＹＴＥ目の
		dw  	setop_adr
	に反映されます。サウンドコントロールプログラムおよびエディタは、この
	データからＳ．Ｅ．のスタートアドレスを知ることができます。

	　＜ＰＲＪ＞ＴＢ．ＳＲＣはソングファイルとリンクされ、＜ＰＲＪ＞ＳＮ
	Ｇ．ＨＥＸ、および＜ＰＲＪ＞ＢＮＫ．ＨＥＸをつくります。したがってＡ
