# Trails Database: Schema Reference

This document specifies the Metadata layer registry schema and the Atlas layer page class definitions.

---

## Part 1: Metadata Layer — Registry Implementation

The Metadata layer is implemented through eight SQLite tables in `retrieval/index/trails.db`. These registries are implementation machinery. They are not the project's top-level identity — the project is a two-layer canon atlas; the registries are how the Metadata layer is built.

### 1.1 Registry Table Summary

| Table | Purpose | Standard ID Prefix |
| :--- | :--- | :--- |
| `media_registry` | Umbrella catalog (games, anime, manga, drama CDs) | `media:` |
| `games_registry` | Game-specific arc/platform data (subtype of media) | linked to `media:` |
| `entity_registry` | Characters, factions, places, staff, quests, items, concepts | `char:`, `faction:`, `loc:`, `staff:`, `quest:`, `item:`, `event:`, `concept:` |
| `source_registry` | Ingestion provenance — all ingest origins tracked here | `wiki:`, `official:`, `system:`, `local:` |
| `appearance_registry` | Many-to-many entity↔media links | |
| `relationship_registry` | Typed edge layer across all entity types | |
| `chunk_registry` | Textual storage — bios, lore, scripts — with FTS5 index | `raw:`, `curated:`, `script:` |
| `lifecycle_registry` | Editorial state tracking per entity | |

### 1.2 Identifier Standards

- **Characters**: `char:estelle_bright`, `char:van_arkride`
- **Media**: `media:sky_fc`, `media:northern_war`, `media:drama_future`
- **Factions**: `faction:bracer_guild`, `faction:ouroboros`
- **Staff**: `staff:toshihiro_kondo`, `staff:daisuke_ono`
- **Locations**: `loc:crossbell_city`, `loc:grancel`

The `games_registry` is a subtype view linked to `media_registry`. Games do not maintain a separate primary ID.

### 1.3 Relationship Types

The `relationship_registry` provides typed edges across all entity types:

| Type | Connects |
| :--- | :--- |
| `voiced_by_jp` / `voiced_by_en` | Character → Staff (voice actor) |
| `member_of` | Character/Staff → Faction |
| `published_by` | Media → Company |
| `designed_by` | Staff → Character |

### 1.4 Source Trust Tiers

The `source_registry` tracks all ingest origins with a `trust_tier` integer:

| Tier | Meaning |
| :--- | :--- |
| 0 | Official (Falcom, XSEED, NIS America) |
| 1 | Wikipedia (EN or JA) |
| 2 | Fan wiki (Kiseki Fandom, Miraheze) |
| 3 | Speculative / unverified |

### 1.5 Chunk Types (Phase 21 normalized vocabulary)

The `chunk_registry` holds textual content typed by `chunk_type`. As of Phase 21, the vocabulary is normalized to 8 types:

| Type | Description |
| :--- | :--- |
| `background` | Character biography, origin, context (consolidates raw_bio, curated_bio, clean_bio, bio) |
| `lead` | 1–3 sentence overview / intro line |
| `dialogue` | Quotes, stage directions, narration |
| `lore` | World-building, tropes, meta context (consolidates lore_expanded, series_lore, character_tropes) |
| `appearance` | Physical description |
| `personality` | Character trait analysis |
| `history` | Arc-specific event history |
| `timeline` | Chronological event entries |

Language column: `en` (default) or `ja` (JA Wikipedia substrate content).

`translated_content` column (TEXT, nullable): English translation of JA chunks, produced by
`scripts/translation/translate_ja_chunks.py`. NULL = not yet translated. When populated, this
column is used by `generate_atlas.py` in place of the raw JA `text_content`, making JA
Wikipedia's deeper coverage available to the EN synthesis prompt. Tagged as `[JA Wikipedia]`
in Atlas section labels.

### 1.6 Lifecycle States

The `lifecycle_registry` tracks editorial progress per entity:
`raw` → `normalized` → `curated` → `export_ready`

## Part 1b: Phase 21 — Additional Tables

Four tables were added in Phase 21 to improve LLM comprehension and traversal:

### entity_summary
Synthesized Atlas entries and completeness scores.

```sql
CREATE TABLE entity_summary (
    entity_id    TEXT PRIMARY KEY REFERENCES entity_registry(entity_id),
    summary      TEXT,           -- 2-4 paragraph Atlas prose (generated by Claude)
    completeness INTEGER,        -- 0-100: min(chunk_count * 20, 100)
    updated_at   TEXT
)
```

### relationships
Typed directed graph edges between entities.

```sql
CREATE TABLE relationships (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    subject_id   TEXT,
    predicate    TEXT,   -- appears_in | member_of | trained_under | mentor_of | rivals |
                         --   allied_with | child_of | parent_of | sibling_of
    object_id    TEXT,
    context_arc  TEXT,
    spoiler_band INTEGER DEFAULT 0,
    UNIQUE (subject_id, predicate, object_id, context_arc)
)
```

Current edge counts: 4,601 total (3,422 appears_in · 837 member_of · 117 child_of · 88 parent_of · 64 mentor_of · 49 trained_under · 20 rivals · 4 family)

### entity_appearances
Per-entity game coverage flags with role classification.

```sql
CREATE TABLE entity_appearances (
    entity_id TEXT,
    game_id   TEXT,
    role_type TEXT DEFAULT 'supporting',  -- protagonist | playable | supporting | antagonist | mentioned
    PRIMARY KEY (entity_id, game_id)
)
```

### chunk_mentions
Cross-entity provenance — which chunks mention which entities by name.

```sql
CREATE TABLE chunk_mentions (
    chunk_id            TEXT,
    mentioned_entity_id TEXT,
    PRIMARY KEY (chunk_id, mentioned_entity_id)
)
```
80,594 rows — enables "find all chunks that mention Estelle" queries without full-text search.

## Part 2: Atlas Layer — Page Class Definitions

The Atlas layer is organized into six page classes. Each class maps to Metadata registry data and carries a long-form explanatory prose section.

In MediaWiki, each class is backed by a Cargo table definition, a Page Form, and a Scribunto infobox module. See `docs/mediawiki_plan.md` for implementation.

### 2.1 Page Classes

| Class | Maps to Registry | Primary entity_type values |
| :--- | :--- | :--- |
| Character | `entity_registry` | `char` |
| Place | `entity_registry` | `loc` |
| Organization | `entity_registry` | `faction` |
| Event | `entity_registry` | `event` |
| Concept | `entity_registry` | `concept`, `item` |
| Media | `media_registry` | (all media types) |

### 2.2 Character Page Fields (Metadata section)

| Field | Type | Notes |
| :--- | :--- | :--- |
| `entity_id` | String | Stable ID from `entity_registry` (e.g., `char:estelle_bright`) |
| `name_en` | String | Canonical English display name |
| `name_ja` | String | Japanese name (kanji/kana) |
| `aliases` | List | All known alternate names/transliterations |
| `arc_first_appearance` | String | Arc of first canon appearance |
| `spoiler_band` | Integer | 10–100; governs retrieval safety |
| `affiliation` | List | Faction IDs from `faction:` namespace |
| `voice_jp` | String | `staff:` ID of JP voice actor |
| `voice_en` | String | `staff:` ID of EN voice actor |

### 2.3 Place Page Fields (Metadata section)
