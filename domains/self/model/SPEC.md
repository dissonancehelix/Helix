## 1. DOMAIN SCOPE

The Self domain operates on **agent state trajectory data** — sequences of
states generated by human participants, AI systems, or controlled simulated agents.

The domain's **process model** is trajectory dynamics:
```
possibility space → constraint → tension → collapse/commitment → new trajectory
```

Within that process model, **DCP (Decision Compression Principle)** is the event-level
principle this domain operationalizes — identifying, classifying, and probing
collapse/commitment events in agent trajectories.

**Collapse morphology** (`core/engine/compute/invariants/dcp/morphology.py`) is the classification
layer for post-collapse outcomes.

Its goal is to extract structural invariants in how agents traverse constrained
possibility spaces, with emphasis on:

- Pre-collapse dynamics (tension accumulation, constraint intensification)
- Collapse events (commitment points, sharp possibility-space reduction)
- Post-collapse trajectory class (morphology of outcome)
- Perturbation response (recovery shape after external disruption)

**Out of scope** for this domain:
- Consciousness theory or qualia claims
- Metaphysical ontology of self or experience
- Proof of any specific cognitive mechanism

## 2. DOMAIN-LOCAL STRUCTURAL SIGNALS

These signals are extracted from agent trajectory data and are domain-local.
They are NOT HelixEmbedding axes. They feed the projection stage (Stage 7),
which produces a `CognitionStateVector` that is then projected into the shared embedding.

All signals below are provisional operational definitions subject to calibration.

### 2.1 Trajectory State Signals

**`possibility_breadth`** (`float`, `[0, 1]` normalized)  
Estimated proportion of the agent's state space that is accessible from the current
state. 1.0 = fully unconstrained; 0.0 = singular forced transition.  
*Operationalization*: task-dependent. For branching agents: `branching_factor / max_branching_factor`.
For AI systems: token-probability entropy normalized by model vocabulary size.

**`constraint_intensity`** (`float`, `≥ 0`)  
Rate of change of `possibility_breadth` per unit time. Positive values indicate
constraint accumulation; negative values indicate expansion.  
*Operationalization*: `Δpossibility_breadth / Δt` over a sliding window.

**`tension_level`** (`float`, `[0, 1]`)  
Accumulated unresolved constraint. Proxy for pre-collapse pressure.  
*Operationalization*: cumulative integral of `constraint_intensity` over the
current trajectory segment, normalized by observed maximum.

### 2.2 Collapse Event Signals

**`collapse_threshold`** (`float`, `[0, 1]`)  
The `possibility_breadth` value at which a collapse event is declared.  
*Status*: Not calibrated. Provisional default: `0.10` (10% of maximum breadth).

**`collapse_magnitude`** (`float`, `[0, 1]`)  
Fractional reduction in `possibility_breadth` across the collapse event window.  
`collapse_magnitude = (breadth_pre - breadth_post) / breadth_pre`

**`collapse_morphology`** (`enum` — `CollapseMorphology`)  
Post-collapse trajectory class. Use canonical values from `core/engine/compute/invariants/dcp/morphology.py`:

| Value | Coherence Effect | Description |
|-------|-----------------|-------------|
| `TRANSFORMATIVE` | Increases | New stable attractor; structurally distinct from pre-collapse |
| `DISSOLUTIVE` | Decreases | Trajectory destabilizes; no new coherent structure |
| `CIRCULAR` | Restored to prior | Return toward pre-collapse attractor |
| `DEFERRED_SUSPENDED` | Intermediate | Sustained high-tension; collapse boundary not crossed |

Classification basis: trajectory similarity metric (cosine similarity of state-sequence
feature vectors before/after candidate collapse event). Threshold: not calibrated.

### 2.3 Post-Collapse Trajectory Signals

**`post_collapse_narrowing`** (`float`, `[0, 1]`)  
Reduction in trajectory variance in the window following a collapse event,
relative to the pre-collapse window.

**`perturbation_response`** (`float`, `[-1, 1]`)  
Signed change in `possibility_breadth` following external perturbation. Negative
= further constraint; positive = expansion (recovery).

### 2.4 Research-Stage Variable (placeholder only)

**`self_boundary_stability`**  
Stability of the agent's distinction between self-state and environment-state.
No operationalization exists yet. Included as a named placeholder for future probing.

## 3. SHARED EMBEDDING PROJECTION

Self domain signals are fused into a `CognitionStateVector` and projected
into the shared `HelixEmbedding` format via Stage 7.

**Provisional projection mapping** (not yet implemented):

| HelixEmbedding Axis | Primary Cognition Signal | Notes |
|---------------------|--------------------------|-------|
| `structure` | `collapse_morphology` (encoded) | Morphology class as structural regularity proxy |
| `complexity` | `tension_level` | Accumulated constraint as complexity proxy |
| `repetition` | `post_collapse_narrowing` | Trajectory convergence as repetition proxy |
| `density` | `constraint_intensity` | Rate of constraint change |
| `variation` | `possibility_breadth` | Open state space as variation proxy |
| `expression` | `perturbation_response` | Recovery shape |

**Non-equivalence rule**: HelixEmbedding axis names are shared system-wide.
Cognition signal names are domain-local. The mapping above is explicit and
provisional — it is not naming equivalence.

**Projection schema**: `cognition_v0` (not yet stamped on any artifact).

## 6. VALIDATION RULES

The following rules are defined but not yet implemented in any validation harness:

- **Trajectory continuity**: `possibility_breadth` series must not jump by more than
  `0.30` per step without a declared event boundary
- **Collapse detection consistency**: re-running collapse detection on a fixed input
  must produce identical event lists (deterministic)
- **Morphology calibration**: morphology labels must have a calibration basis —
  classification without a null corpus comparison is not sufficient for promotion
- **Null model guard**: a random-walk agent (no structural constraint) must not
  produce collapse events at a rate above the domain null rate threshold
  (not yet established)

## 7. METRIC SPACE / SIMILARITY / DISTANCE

Cognition embeddings use the same shared HelixEmbedding metric:

```
distance(a, b) = euclidean(a, b) / sqrt(6)    ∈ [0, 1]
similarity(a, b) = 1 - distance(a, b)          ∈ [0, 1]
```

For domain-local `CognitionStateVector` comparison (before projection), Euclidean
distance in the 7-axis domain-local space applies. Normalization factor: `sqrt(7)`.

Triangle inequality applies to distance, not similarity. Violations are
`STRUCTURAL_FAILURE` and must be flagged before Atlas promotion.

## 10. PROMOTION CONDITIONS

Collapse event candidates must pass the global 6-criterion gate defined in
`docs/GOVERNANCE.md`:

1. Reproducibility — ≥ 2 independent runs on same agent data
2. Multi-domain observation — ≥ 2 domains observe the same invariant class
3. Minimum confidence ≥ domain floor
4. Pass rate ≥ 80% across probed sessions
5. Signal above null-model threshold
6. Latest probe version used at time of submission

No cognition-specific tightening of these criteria is defined yet.

## 11. KNOWN ASSUMPTIONS / OPEN CONSTRAINTS

- `possibility_breadth` is assumed to be estimable from external behavioral data
  without direct access to the agent's internal state. This assumption is strong
  for toy agents and weak for opaque AI systems.
- Collapse morphology classification assumes trajectory data is available in a
  window both before and after the collapse event. For causal/online systems this
  requires buffering.
- `self_boundary_stability` is listed as a domain variable but has no
  operationalization. It should not be included in any fixture until a concrete
  measurement protocol is defined.
- The provisional projection mapping (§3) is a hypothesis — the specific signal-
  to-axis assignments have not been validated against cross-domain comparison data.
- Collapse morphology categories are not mutually exclusive in all edge cases.
  A `CIRCULAR` trajectory may also exhibit `DISSOLUTIVE` transients. Handling of
  overlapping classifications is not yet defined.
