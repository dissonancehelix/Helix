# Aesthetics

**Status:** placeholder domain capsule  
**Role:** decompressed domain room for color, atmosphere, enclosure, texture, environments, ruins, and inhabited visual fields  
**Parent map:** `DISSONANCE.md`  
**Not role:** raw evidence dump, proof file, generic personality label, or replacement for the main map

## What this domain is to the operator

Aesthetics is a first-class experience domain because visual atmosphere, color, darkness, warmth, enclosure, texture, ruins, and earthy/Mesoamerican motifs reveal what makes an environment feel inhabitable.

## What it is not

- Not a moodboard dump.
- Not a color-palette preference list.
- Not generic dark/cozy aesthetics.
- Not proof without examples and negative controls.

## Core reward mechanics

- Warmth, darkness, and enclosure.
- Texture and material residue.
- Ruins, old spaces, and lived-in environments.
- Mesoamerican/earthy motifs.
- Inhabited environment over empty decoration.

## Core rejection mechanics

- Atmospheric surface without re-entry.
- Clean visual style that removes residue.
- Generic darkness without warmth or structure.
- Decorative motifs without an inhabitable field.

## Evidence anchors

- Placeholder only. Add curated anchors after backgrounds, photographs, screenshots, and visual archives are reviewed.

## False positives / negative controls

- Dark but sterile.
- Earthy but flat.
- Detailed but not inhabitable.

## How this domain feeds the global cognitive map

This domain tests interiority, enclosure, residue, field texture, and recognition/taste prediction outside explicitly narrative or game domains.

## Best compression

> Aesthetic pull comes from inhabitable atmosphere, not decoration alone.
