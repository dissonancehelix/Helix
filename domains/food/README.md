# Food

**Status:** placeholder domain capsule  
**Role:** decompressed domain room for food, drink, texture, ritual, comfort, and bounded abundance  
**Parent map:** `DISSONANCE.md`  
**Not role:** raw evidence dump, proof file, generic personality label, or replacement for the main map

## What this domain is to the operator

Food is a first-class experience domain because taste, texture, comfort, seasonal ritual, and abundance limits expose reward mechanics that are not reducible to simple favorite foods.

## What it is not

- Not a recipe dump.
- Not generic comfort-food labeling.
- Not nutrition tracking.
- Not a substitute for sensory or body domains.

## Core reward mechanics

- Bounded abundance.
- Soft base / sharp event.
- Seasonal ritual.
- Cold/creamy comfort.
- Texture layering.

## Core rejection mechanics

- Unbounded mess or excess.
- Texture collapse.
- Novelty without comfort or return.
- Flattening ritual into calorie/preference facts.

## Evidence anchors

- Placeholder only. Add curated anchors after food/drink evidence is reviewed.

## False positives / negative controls

- Rich but texture-flat food.
- Abundant but unbounded food.
- Seasonal object without ritual charge.

## How this domain feeds the global cognitive map

This domain tests bounded abundance, layered contrast, ritual return, and sensory comfort as taste mechanics.

## Best compression

> Food works when comfort has structure and abundance has bounds.
