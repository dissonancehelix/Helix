# Wiki Domain

## Domain Role

Wiki is a first-class externalized cognition domain.

This domain is not just "Wikipedia editing" as an activity. It is a public knowledge chamber the operator has lived inside since 2012: article structure, citation logic, template systems, infoboxes, Wikidata records, Commons media, talk-page negotiation, and future-reader inference.

Wikipedia turns private pattern sense into public architecture. A good edit makes a subject easier to enter, verify, update, and reuse.

Best compression:

> Wiki is public knowledge architecture under maintenance pressure.

## LLM Reading Guidance

If this README is shared with an LLM, it should infer:

- The operator edits through object attachment, not detached productivity.
- Repeated edits to the same pages are evidence of return, repair, and stewardship.
- Template and infobox work are not side chores; they are public interface design.
- Wikidata and Commons work show a preference for addressable, reusable knowledge structure.
- The account should not be reduced to fandom. The core pattern is public structure maintenance around subjects that matter enough to re-enter.
- The wiki domain is a lived external cognitive tool: a place where attention becomes durable public architecture.

## False Positives

- Treating edit count as the main signal.
- Treating subject matter as mere fandom.
- Treating template work as technical fussing instead of future-reader interface design.
- Treating small edits as low-value when they preserve continuity, currency, or reuse.
- Treating public editing as social performance rather than object-mediated repair.
