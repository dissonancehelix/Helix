# Coordinate vs Ontology Separation Protocol

## 1. Definition of Terms
- **Projection/Manifold (Coordinate System)**: The geometric representation or mapping of domain data into a lower-dimensional space (e.g., SVD, PCA, UMAP).
- **Bases (Ontology)**: The fundamental structural primitives (Ring 0) and elements (Ring 1) that define the system's identity and mechanism.

## 2. Separation Rules
- **Coordinate Collapse != Ontological Reduction**: A discovery that a manifold is low-rank (e.g., k_90 = 1) is a coordinate convenience, not a justification for reducing the number of ontological bases.
- **Coordinate Isolate != Ontological Debt**: A domain's isolation in a manifold projection does not automatically imply the need for a new base (ontology mutation).
- **Mutation Justification**: Only the combination of cross-family invariance (stability under hostility) and transfer resolution justification warrants ontology mutation.

## 3. Direction of Influence
Ontology informs coordinates. If a new base improves coordinate resolution while maintaining invariance, the ontology is expanded. Coordinate artifacts (e.g., scaling-dependent clusters) never inform ontology.
