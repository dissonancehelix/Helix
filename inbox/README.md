# Inbox

`inbox/` is the operator drop zone.

Put unsorted local evidence here: exports, modlists, screenshots, bookmark
files, notes, captures, and other loose material. Agents treat it like a mail
sorting tray:

- read the drop locally,
- extract the relevant signal,
- place durable records in the owning domain,
- place generated interpretation in a report or lab,
- preserve raw provenance under `archive/raw/` when it needs to be retained,
- leave bulky or private dropped files out of Git.

Nothing in `inbox/` is canon by location. It becomes useful only after it is
sorted into a bounded context.
